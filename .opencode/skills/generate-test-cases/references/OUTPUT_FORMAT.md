# 输出格式参考：测试产物

## 1. 项目理解摘要

- 项目类型
- 关键模块
- 对外接口
- 风险较高区域
- 当前已知测试现状
- 主要假设

## 2. 测试策略

建议按层说明：

- 该项目适合哪些测试层
- 哪些最值得先自动化
- 哪些更适合人工探索或验收

## 3. Traceability Matrix

建议使用以下列：

| 设计点 / 模块 | 入口 / 接口 | 风险 | 测试层 | 应测场景 | 优先级 | 备注 |
|---|---|---|---|---|---|---|

## 4. 详细测试用例

每个模块一组，建议用如下字段：

- Case ID
- 标题
- 目标
- 目标对象
- 前置条件
- 输入 / 测试数据
- 步骤
- 预期结果
- 优先级
- 风险覆盖
- 自动化建议

## 5. 建议目录布局

例如：

```text
tests/
  unit/
  integration/
  contract/
  e2e/
  fixtures/
```

## 6. 不确定项

把这些明确列出来：

- 无法从当前仓库确认的行为
- 需要产品/开发确认的边界
- 推断得出的假设
