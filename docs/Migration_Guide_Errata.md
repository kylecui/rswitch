## 📋 用户行动指南

### 如果你正在评估 rSwitch

1. **阅读新增的"模块实现状态"章节**
   - 了解哪些功能现在可用
   - 了解哪些功能在路线图中

2. **检查你的用例是否匹配可用功能**
   - ✅ 可用: L2 交换、基础 VLAN、VOQd QoS
   - ❌ 不可用: ACL、L3 路由、流量镜像、完整 VLAN (PCP/QinQ)

### 如果你已经部署了 rSwitch

1. **验证你使用的模块**
   ```bash
   sudo rswitchctl show-pipeline
   ```

2. **如果使用了未实现的模块**
   - `acl`, `route`, `mirror`, `qos`: 这些模块不存在
   - 配置可能已加载但不会生效
   - 建议简化 profile，只使用已实现的模块

3. **临时解决方案**
   - **需要 ACL**: 使用 iptables/nftables 在内核网络栈层面
   - **需要路由**: 使用 Linux 内核路由表
   - **需要镜像**: 使用 tc mirror action

### 如果你在开发模块

1. **参考"模块实现差距分析"文档**
   - `/home/kylecui/dev/rSwitch/rswitch/docs/Module_Implementation_Gap_Analysis.md`
   - 了解每个缺失模块的详细规范

2. **优先级**
   - P0: acl.bpf.c, mirror.bpf.c, VLAN PCP/DEI 处理
   - P1: route.bpf.c, qos.bpf.c
   - P2: QinQ 支持, stp.bpf.c
